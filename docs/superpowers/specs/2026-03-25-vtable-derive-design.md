# Design: `#[vtable(...)]` System for OpenWA

## Context

OpenWA has 7+ manually-written vtable structs (`PaletteVtable`, `DisplayBaseVtable`, `DDDisplayVtable`, etc.) with hand-coded `#[repr(C)]` layouts, manual `impl` wrapper methods, and ad-hoc vtable patching. The existing `#[derive(FieldRegistry)]` macro and `inventory`-based registry provide a proven pattern for auto-generating struct metadata. This design extends that infrastructure to vtables, providing a single definition per vtable that handles introspection, calling, and replacement.

**Constraints:**
- A C++ class can have multiple vtables at different struct offsets (multiple inheritance)
- The same vtable type can theoretically serve multiple classes
- All vtable methods are `unsafe extern "thiscall"` (ECX = this)
- Target is i686-pc-windows-msvc (32-bit, 4-byte pointers/fn pointers)

## Design

### 1. Vtable Definition (`#[vtable(...)]`)

**Input syntax** — declare only known methods with `#[slot(N)]`:

```rust
#[openwa_core::vtable(size = 5, va = 0x0066_A2E4, class = "Palette")]
pub struct PaletteVtable {
    /// set_mode(this, mode) — called with mode=7 during hardware init
    #[slot(2)]
    pub set_mode: unsafe extern "thiscall" fn(*mut Palette, u32),
    /// init(this) — called during hardware init
    #[slot(3)]
    pub init: unsafe extern "thiscall" fn(*mut Palette),
    /// reset(this) — called first during hardware init
    #[slot(4)]
    pub reset: unsafe extern "thiscall" fn(*mut Palette),
}
```

**Generated output:**

```rust
// 1. The actual #[repr(C)] struct with gap-fillers
#[repr(C)]
pub struct PaletteVtable {
    pub _slot_0: usize,
    pub _slot_1: usize,
    /// set_mode(this, mode) — called with mode=7 during hardware init
    pub set_mode: unsafe extern "thiscall" fn(*mut Palette, u32),
    /// init(this) — called during hardware init
    pub init: unsafe extern "thiscall" fn(*mut Palette),
    /// reset(this) — called first during hardware init
    pub reset: unsafe extern "thiscall" fn(*mut Palette),
}

// 2. Registry integration (introspection)
impl HasVtableRegistry for PaletteVtable {
    fn vtable_info() -> &'static VtableInfo {
        static INFO: VtableInfo = VtableInfo {
            struct_name: "PaletteVtable",
            class_name: "Palette",
            ghidra_va: 0x0066_A2E4,
            slot_count: 5,
            slots: &[
                VtableSlotEntry { index: 2, name: "set_mode", doc: "set_mode(this, mode) ..." },
                VtableSlotEntry { index: 3, name: "init", doc: "init(this) ..." },
                VtableSlotEntry { index: 4, name: "reset", doc: "reset(this) ..." },
            ],
        };
        &INFO
    }
}

inventory::submit!(VtableRegistration {
    info: <PaletteVtable as HasVtableRegistry>::vtable_info(),
});

// 3. Address registry entry (replaces define_addresses! for this vtable)
inventory::submit!(AddrEntry {
    va: 0x0066_A2E4,
    name: "PALETTE_VTABLE",
    kind: AddrKind::Vtable,
    class_name: Some("Palette"),
    calling_convention: None,
});

// 4. Companion binding macro (see Part 2)
#[macro_export]
macro_rules! bind_PaletteVtable { ... }
```

**Proc macro implementation** in `openwa-derive/src/lib.rs`:
- Parse `#[vtable(...)]` attributes for `size`, `va` (optional), `class` (optional)
- Parse `#[slot(N)]` on each field to determine position
- Validate: no duplicate slots, all slots < size, slot indices ascending
- Generate gap-filler fields `_slot_N: usize` for missing indices
- Preserve doc comments, visibility, and field types from input
- Generate compile-time size assertion: `const _: () = assert!(size_of::<VtableName>() == size * 4);`
- Parse fn pointer types to extract `this` type (`*mut T` or `*const T`), params, and return type
- Generate `HasVtableRegistry` impl, inventory submissions, and binding macro
- If `va` is provided: generate `pub const` and `AddrEntry` submission. Name derived from `class` attribute: `"Palette"` → `PALETTE_VTABLE`. If no `class`, derive from struct name: `PaletteVtable` → `PALETTE_VTABLE`
- If `va` is omitted: no `AddrEntry`, no const — vtable exists purely as a type

### 2. Calling Wrappers (`bind_XxxVtable!`)

The derive generates a companion macro named `bind_{VtableName}!` that creates method wrappers:

```rust
// Generated by #[derive(Vtable)] on PaletteVtable:
#[macro_export]
macro_rules! bind_PaletteVtable {
    ($class:ty, $vtable_field:ident) => {
        impl $class {
            #[doc = "set_mode(this, mode) — called with mode=7 during hardware init"]
            pub unsafe fn set_mode(&mut self, mode: u32) {
                ((*self.$vtable_field).set_mode)(self as *mut Self, mode)
            }
            #[doc = "init(this) — called during hardware init"]
            pub unsafe fn init(&mut self) {
                ((*self.$vtable_field).init)(self as *mut Self)
            }
            #[doc = "reset(this) — called first during hardware init"]
            pub unsafe fn reset(&mut self) {
                ((*self.$vtable_field).reset)(self as *mut Self)
            }
        }
    };
}
```

**Usage:**
```rust
// In palette.rs, after struct Palette { vtable: *const PaletteVtable, ... }
bind_PaletteVtable!(Palette, vtable);

// Now you can write:
(*pal).set_mode(7);   // instead of vcall!(pal, set_mode, 7)
(*pal).init();
(*pal).reset();
```

**Multiple vtables (multiple inheritance):**
```rust
// CGameTask has primary vtable at +0x00
bind_CGameTaskVtable!(CGameTask, vtable);

// SoundEmitter is a sub-object with its own vtable — bind on its own struct
bind_SoundEmitterVTable!(SoundEmitter, vtable);
```

**Method signature extraction:** The proc macro parses the fn pointer type as `syn::Type::BareFn` to extract:
- First param is `this` — if `*const T`, generates `&self`; if `*mut T`, generates `&mut self`
- Remaining params become the wrapper's named parameters (e.g., `p0: u32`, `p1: *mut u8`)
- Return type is preserved (default `()` if absent)
- The wrapper calls through `self.$vtable_field` and casts `self` to the correct pointer type

**Type safety:** The binding macro passes `self as *mut Self` (or `*const Self`) as the `this` argument. The `$class` type passed to the macro must match the `this` pointer type in the vtable fn signatures. If mismatched, Rust's pointer cast rules apply silently — the macro does not enforce this, but the mismatch would be visible at the invocation site.

**Note on `&mut self`:** This follows the existing hand-written pattern (e.g., `Palette::reset(&mut self)`). Using `&mut self` creates a mutable reference from what is often a raw pointer, which could theoretically violate aliasing rules. This is acceptable for this project since all callers are already in `unsafe` blocks working with raw game memory, and the `vcall!` macro remains available for raw-pointer-only dispatch.

### 3. Vtable Replacement (`vtable_replace!`)

A declarative macro that expands to executable code for use inside `install()` functions:

```rust
pub fn install() -> Result<(), String> {
    unsafe {
        vtable_replace!(PaletteVtable at va::PALETTE_VTABLE {
            slot set_mode => my_set_mode,
            slot init => my_init,
        })?;
    }
    Ok(())
}
```

**Expanded code** (for `slot 2 => my_set_mode` — pure replace, no original saved):

```rust
{
    let vtable_ptr = $crate::rebase::rb(va::PALETTE_VTABLE) as *mut u32;
    let slot_count = core::mem::size_of::<PaletteVtable>() / 4;
    $crate::vtable::patch_vtable(vtable_ptr, slot_count, |vt| {
        // Slot 2: pure replace
        let slot = vt.add(2);
        *slot = my_set_mode as *const () as u32;
    })
}
```

**Expanded code** (for `slot 3 [originals::INIT] => my_init` — replace + save original):

```rust
{
    let vtable_ptr = $crate::rebase::rb(va::PALETTE_VTABLE) as *mut u32;
    let slot_count = core::mem::size_of::<PaletteVtable>() / 4;
    $crate::vtable::patch_vtable(vtable_ptr, slot_count, |vt| {
        // Slot 3: save original, then replace
        let slot = vt.add(3);
        originals::INIT.store(*slot, core::sync::atomic::Ordering::Relaxed);
        *slot = my_init as *const () as u32;
    })
}
```

**Calling the original:** User defines statics in a visible scope and passes them via `[static]` syntax:

```rust
mod originals {
    pub static INIT: core::sync::atomic::AtomicU32 =
        core::sync::atomic::AtomicU32::new(0);
}

vtable_replace!(PaletteVtable at va::PALETTE_VTABLE {
    slot 2 => my_set_mode,                          // no original needed
    slot 3 [originals::INIT] => my_init,            // save original
})?;

// In the replacement:
unsafe extern "thiscall" fn my_init(this: *mut Palette) {
    log!("init()");
    let orig: unsafe extern "thiscall" fn(*mut Palette) =
        core::mem::transmute(originals::INIT.load(Ordering::Relaxed));
    orig(this);
}
```

The `[static_name]` syntax is optional — if omitted, no original is saved (pure replacement, no call-through). The macro does NOT generate inaccessible statics.

**Slot index resolution:** Slot indices are specified explicitly in the macro call. This is compile-time, simple, and the indices are directly visible in the vtable definition above.

**Size derivation:** The `size` parameter is NOT required — the macro computes it as `size_of::<VtableType>() / 4` from the vtable struct type, which already encodes the correct size via `#[vtable(size = N)]`.

Revised syntax:
```rust
vtable_replace!(PaletteVtable at va::PALETTE_VTABLE {
    slot 2 => my_set_mode,                         // pure replace, original discarded
    slot 3 [originals::INIT] => my_init,           // replace + save original to static
})?;
```

### 4. New Types in `openwa-core::registry`

```rust
/// Metadata for a single vtable slot
pub struct VtableSlotEntry {
    pub index: u32,
    pub name: &'static str,
    pub doc: &'static str,
}

/// Metadata for a vtable struct
pub struct VtableInfo {
    pub struct_name: &'static str,
    pub class_name: &'static str,
    pub ghidra_va: u32,
    pub slot_count: u32,
    pub slots: &'static [VtableSlotEntry],
}

impl VtableInfo {
    pub fn slot_by_name(&self, name: &str) -> Option<&VtableSlotEntry> { ... }
    pub fn slot_by_index(&self, index: u32) -> Option<&VtableSlotEntry> { ... }
}

/// Trait implemented by #[derive(Vtable)] structs
pub trait HasVtableRegistry {
    fn vtable_info() -> &'static VtableInfo;
}

/// Inventory collection wrapper
pub struct VtableRegistration {
    pub info: &'static VtableInfo,
}
inventory::collect!(VtableRegistration);

/// Query API
pub fn vtable_info_for(struct_name: &str) -> Option<&'static VtableInfo> { ... }
pub fn all_vtable_info() -> impl Iterator<Item = &'static VtableInfo> { ... }
```

### 5. `vtable::patch_vtable` Helper

Move/generalize the existing vtable patching logic from `display.rs` into a shared utility:

```rust
// openwa-core::vtable (new module)

/// Temporarily makes a vtable writable, calls the patcher, restores protection.
/// Returns Err if VirtualProtect fails.
pub unsafe fn patch_vtable(
    vtable_ptr: *mut u32,
    slot_count: usize,
    patcher: impl FnOnce(*mut u32),
) -> Result<(), String> {
    let size = slot_count * 4;
    let mut old_protect = 0u32;
    if VirtualProtect(vtable_ptr as _, size, PAGE_READWRITE, &mut old_protect) == 0 {
        return Err("VirtualProtect failed".into());
    }
    patcher(vtable_ptr);
    VirtualProtect(vtable_ptr as _, size, old_protect, &mut old_protect);
    Ok(())
}
```

This already exists in `display.rs` — factor it out.
